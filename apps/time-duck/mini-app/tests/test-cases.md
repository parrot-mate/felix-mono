# Time Duck 微信小程序 提醒助手 测试用例

## Test Case: 小程序可以启动并展示主体验

- Scenario: 用户首次打开 Time Duck 小程序 demo。
- Preconditions: 小程序项目已在微信开发者工具中导入，默认进入主体验。
- Steps:
  1. 打开 `felix-mono/apps/time-duck/mini-app`。
  2. 等待首页渲染完成。
  3. 查看页面标题、顶部设备状态、语音交互入口和底部 tab。
- Expected Result: 页面正常渲染，顶部展示当前页面标题“报时”和轻量设备状态入口，主体展示语音入口和浅色大黄鸭主题界面；默认停留在语音交互页，没有报错、白屏或布局溢出。

## Test Case: 用户可以在语音页和提醒页之间切换

- Scenario: 用户通过底部 tab 切换核心功能区域。
- Preconditions: 页面已打开，默认停留在语音交互页。
- Steps:
  1. 查看底部 tab 的“报时”和“提醒”入口。
  2. 点击“提醒”。
  3. 查看提醒列表和空状态。
  4. 点击“报时”返回语音页。
- Expected Result: 底部 tab 选中态清晰；语音页只承载语音输入和最近创建提醒；提醒页直接展示提醒列表空状态。

## Test Case: 用户可以完成 蓝牙连接

- Scenario: 用户连接提醒设备。
- Preconditions: 页面已打开，蓝牙状态为未连接。
- Steps:
  1. 点击顶部设备状态入口。
  2. 观察搜索中状态。
  3. 观察连接中状态。
  4. 等待 连接成功。
- Expected Result: 顶部设备状态入口展示已连接状态，用户仍停留在当前页面，蓝牙状态不会作为大卡片占据主体内容区。

## Test Case: 蓝牙连接失败可以重试

- Scenario: 蓝牙连接失败后用户重试。
- Preconditions: 页面已打开，可触发蓝牙连接失败。
- Steps:
  1. 点击顶部设备状态入口。
  2. 触发连接失败。
  3. 查看顶部设备状态切换为重试连接。
  4. 再次点击顶部设备状态入口并触发连接成功。
- Expected Result: 失败状态清晰可见但不抢占主体区域；重试后状态可以恢复到搜索中/连接中，并最终进入已连接状态。

## Test Case: 创建提醒后提醒页展示任务列表

- Scenario: 用户创建提醒后进入提醒页查看列表。
- Preconditions: 页面已打开，初始提醒数据为空，用户已通过语音成功创建至少一条提醒。
- Steps:
  1. 打开首页。
  2. 完成一次语音创建提醒。
  3. 点击底部 tab 的“提醒”。
  4. 检查任务标题、提醒时间、重复/提醒状态和完成按钮。
- Expected Result: 任务列表按卡片行展示，信息层级清晰，任务状态可区分，底部 tab 不会遮挡最后一条任务。

## Test Case: 用户可以开始并取消语音录入

- Scenario: 用户误触或临时取消语音输入。
- Preconditions: 页面已打开，语音按钮处于空闲状态。
- Steps:
  1. 按住语音按钮。
  2. 观察录音中状态、声波和计时。
  3. 点击取消。
- Expected Result: 页面进入录音中状态后可以取消；取消后回到空闲状态，不创建新任务。

## Test Case: 用户可以通过语音创建明确时间的待办

- Scenario: 用户说出包含明确时间的提醒。
- Preconditions: 页面已打开，语音结果为“明早八点提醒我带电脑”。
- Steps:
  1. 按住语音按钮开始录音。
  2. 完成录音。
  3. 等待语音识别状态结束。
  4. 等待 AI 解析步骤完成。
  5. 等待提醒开启完成。
  6. 确认结果。
  7. 切换到底部 tab 的“提醒”页。
- Expected Result: AI 解析面板展示任务标题“带电脑”和明确提醒时间；提醒开启后，新任务出现在语音页的最近创建区域，并进入提醒页列表且展示待提醒状态。

## Test Case: 用户可以创建重复提醒任务

- Scenario: 用户说出带重复规则的提醒。
- Preconditions: 语音结果为“每天晚上十点提醒我泡脚”。
- Steps:
  1. 完成一次语音输入。
  2. 等待 AI 解析完成。
  3. 查看解析结果。
  4. 确认创建任务。
- Expected Result: 解析结果展示任务标题、晚上十点的提醒时间和每天重复规则；任务进入列表后能看到重复提醒标识。

## Test Case: AI 无法判断时间时任务进入待补时间状态

- Scenario: 用户语音中没有可判断的时间。
- Preconditions: 语音结果为“提醒我整理一下桌面”，AI 返回 `needsManualTime`。
- Steps:
  1. 完成一次语音输入。
  2. 等待 AI 解析完成。
  3. 查看解析结果和任务状态。
- Expected Result: 页面提示无法确定提醒时间；任务可以进入“待补时间”状态，不应展示已设置闹钟成功。

## Test Case: 模拟设置闹钟失败后可以重试

- Scenario: AI 已解析成功，但 native 设置闹钟 失败。
- Preconditions: 可触发 闹钟设置失败。
- Steps:
  1. 完成一次语音输入并等待 AI 解析成功。
  2. 触发闹钟设置失败。
  3. 查看任务状态。
  4. 点击重试设置闹钟。
- Expected Result: 任务保留在列表中，状态为设置失败；重试成功后状态切换为待提醒。

## Test Case: 用户可以查看任务详情

- Scenario: 用户查看已创建任务的完整信息。
- Preconditions: 列表中存在至少一条待提醒任务。
- Steps:
  1. 点击任务卡片。
  2. 查看任务详情面板。
  3. 检查标题、原始语音、提醒时间、重复规则和提醒状态。
  4. 关闭详情面板。
- Expected Result: 详情面板信息完整；用户不能手动把待提醒改为已提醒，列表布局不跳动。

## Test Case: 小屏宽度下页面不溢出

- Scenario: 在窄屏手机尺寸下查看 demo。
- Preconditions: 微信开发者工具或预览环境设置为约 `320px` 宽度。
- Steps:
  1. 打开首页。
  2. 查看顶部状态区、任务卡片和底部语音按钮。
  3. 打开 AI 解析面板。
  4. 打开任务详情面板。
- Expected Result: 文字、按钮、卡片和弹层不横向溢出、不互相遮挡；长标题最多两行省略，详情面板可展示完整文本。

## Test Case: 浅色大黄鸭视觉层级符合设计提案

- Scenario: 检查 UI 是否符合已确认设计方向。
- Preconditions: 小程序可以正常预览。
- Steps:
  1. 查看首页背景、卡片、文字和状态色。
  2. 查看语音录音状态。
  3. 查看 AI 解析弹层。
  4. 查看蓝牙连接状态。
- Expected Result: 页面呈现浅色、圆润、轻快的大黄鸭提醒工具感；主文字清晰，辅助文字不过暗，录音和状态色突出但不刺眼。
